# FreshCart Grocer

A quick-commerce grocery delivery web app — customers browse categories and products, add to cart, and place orders. Admins manage products and view sales reports.

## Run & Operate

- `pnpm --filter @workspace/freshcart run dev` — run the FreshCart web app (port 20100)
- Required env: `VITE_SUPABASE_URL`, `VITE_SUPABASE_PUBLISHABLE_KEY` — Supabase project credentials

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite + TanStack Router (file-based routing)
- Auth & DB: Supabase (PostgreSQL, Row-Level Security)
- UI: shadcn/ui + Tailwind CSS v4
- State: TanStack Query

## Where things live

- `artifacts/freshcart/src/routes/` — all pages (file-based routing via TanStack Router)
- `artifacts/freshcart/src/routes/_authenticated/` — auth-guarded pages (account, orders, checkout, admin)
- `artifacts/freshcart/src/integrations/supabase/` — Supabase client + generated types
- `artifacts/freshcart/src/components/` — shared UI components
- `artifacts/freshcart/src/lib/` — checkout.ts, reports.ts, format.ts

## Architecture decisions

- Migrated from TanStack Start (SSR) to plain Vite — all server functions replaced with direct Supabase client calls
- Auth guard lives in `_authenticated/route.tsx` — checks `supabase.auth.getSession()` and redirects unauthenticated users to `/auth`
- Route tree is auto-generated by `@tanstack/router-plugin` Vite plugin on dev server start (`routeTree.gen.ts`)
- Checkout is COD-only (Razorpay server-side integration was removed; no server keys available)
- `VITE_SUPABASE_PUBLISHABLE_KEY` is the anon key (Supabase renamed it; same value as `VITE_SUPABASE_ANON_KEY`)

## Product

- Homepage with hero, category grid, and featured products
- Category browsing (`/c/:slug`), product detail (`/p/:slug`), search
- Shopping cart with coupon codes
- Authenticated: checkout (COD), order history, account page
- Admin panel: product management, sales reports with CSV export

## Gotchas

- `routeTree.gen.ts` is auto-generated by Vite on startup — do not edit it manually
- Supabase types live in `src/integrations/supabase/types.ts` — regenerate from Supabase dashboard if schema changes
- The `head:` property in route configs is silently ignored in Vite mode (no SSR)
